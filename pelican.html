Archlinux下使用Pelican和Github Pages搭建个人博客

Post in 二 09 六月 2015 |Tags Linux

缘起

虽网龄不小，当博客大行其道之时，我冷眼旁观，只因自身文学修养不高，更关键的是不知要写什么。自从玩了linux系统后，也积累了些笔记，算是有资本可以写写了。另外加上自己的一些其它爱好，如有自己的博客可以练练也是件开心的事。

前不久在linuxtoy上看到网站从WordPress迁移到了pelican，遂好奇心起，研究了下，发现pelican与GitHub配合可以建个人博客，于是就有了本文。

搭建pelican环境

我用的linux发行版为archlinux，有强大的软件仓库和aur库。用yaourt里安装:

yaourt -S pelican pelican-themes-git fabric python-virtualenv  python-virtualenvwrapper python-markdown

当然，如果你的linux发行版不是archlinux或者用的是windows的话，也可以安装pip, 并用pip安装相应的软件。

接下来通过修改.bashrc申明virtualenvwrapper环境:

# virtualenvwrapper config
export PROJECT_HOME=~/dev
export WORKON_HOME=~/dev/virtualenvs
source /usr/bin/virtualenvwrapper.sh

重新打开终端，或执行source .bashrc, 然后执行如下命令:

$ mkproject duncanlock.net-pelican

终端会显示类似的:

New python executable in duncanlock.net-pelican/bin/python
Installing distribute.........done.
Installing pip...............done.
virtualenvwrapper.user_scripts creating
[...]
Creating /home/duncan/dev/duncanlock.net-pelican
Setting project for duncanlock.net-pelican to /home/duncan/dev/duncanlock.net-pelican

这样你就拥有一个虚拟环境~/dev/virtualenvs/duncanlock.net-pelican及一个新文件夹~/dev/duncanlock.net-pelican，用于放置你的项目文件。当你在这虚拟环境时，终端提示符也会改变，(duncanlock.net-pelican)会出现在提示符最前面。

当下次要进入虚拟环境时，可以用workon duncanlock.net-pelican命令进入。

运行pelican

激动的时刻要到了，我们先来运行下pelican-quickstart

Welcome to pelican-quickstart v3.2.0.

This script will help you create a new Pelican-based website.

Please answer the following questions so this script can generate the files needed by Pelican.

Using project associated with current virtual environment. Will save to:
/home/duncan/dev/duncanlock.net-pelican

按提示操作，这里不再赘述了。

操作完成后，可以在duncanlock.net-pelican/content目录下新建一个test.md文件测试下:

Title: Archlinux下使用Pelican和Github Pages搭建个人博客
Date: 2015-06-09 15:56:12
Authors: Bruce Zhou
Category: Linux
Tags: Linux
Slug: pelican

This is a test.

然后运行:

$ make html

或 

$ fab build

个人倾向于用fab，因为arch下的软件包有点问题，当执行make serve时报错，疑是python版本问题，2和3建http server的用法不一样。

执行:

$ fab serve

现在可以用浏览器打开http://localhost:8000来看实际效果。

关于pelican的文档请参考官网.
